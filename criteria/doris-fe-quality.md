# Doris FE (Java) 代码质量标准

> 用于 Evaluator Agent 评估 FE 侧 Java 代码修改的质量

---

## 评分维度

### 1. 编译正确性 (Compilation Correctness)

**权重**: 0.30 | **阈值**: 10/10 (不可通融)

| 检查项 | 命令 | 通过条件 |
|--------|------|----------|
| 编译通过 | `cd fe && mvn package -pl fe-core -DskipTests` | 返回 BUILD SUCCESS |
| 无编译警告(新增) | `mvn compile` 输出检查 | 不引入新的编译警告 |
| 依赖无冲突 | `mvn dependency:tree` | 无版本冲突警告 |

### 2. 单元测试 (Unit Tests)

**权重**: 0.25 | **阈值**: 8/10

| 检查项 | 评分标准 |
|--------|----------|
| 现有测试无回归 | 所有原有测试仍通过: 10分 |
| 新增/修改的公共方法有测试 | 覆盖率 > 80%: 10分, > 60%: 8分, > 40%: 6分, < 40%: 4分 |
| 边界条件测试 | 有 null/空/异常输入测试: 10分, 部分覆盖: 7分, 无: 4分 |
| 异常路径测试 | 异常分支有测试: 10分, 部分覆盖: 7分, 无: 4分 |

### 3. API 兼容性 (API Compatibility)

**权重**: 0.15 | **阈值**: 8/10

| 检查项 | 评分标准 |
|--------|----------|
| SQL 语法兼容 | 旧语法仍可用: 10分, 有 deprecation 提示: 8分, breaking: 0分 |
| REST API 兼容 | 接口签名不变: 10分, 新增字段: 9分, 移除字段: 2分 |
| 配置兼容 | 新配置有默认值: 10分, 必须手动配置: 4分 |
| Thrift/gRPC 兼容 | 无 breaking change: 10分, 仅新增字段: 8分, 修改已有字段: 2分 |

### 4. 架构对齐 (Architecture Alignment)

**权重**: 0.15 | **阈值**: 7/10

| 检查项 | 评分标准 |
|--------|----------|
| 类放在正确的包下 | 逻辑正确: 10分, 基本合理: 7分, 不合理: 3分 |
| 遵循现有设计模式 | Visitor/Builder/Factory 等模式一致: 10分, 轻微偏差: 7分, 不一致: 3分 |
| Nereids vs 旧优化器 | 新功能在 Nereids 实现: 10分, 两者都实现: 8分, 仅旧优化器: 4分 |
| 模块边界清晰 | 无跨模块直接依赖: 10分, 有但合理: 7分, 循环依赖: 0分 |

### 5. 代码风格 (Code Style)

**权重**: 0.05 | **阈值**: 6/10

| 检查项 | 评分标准 |
|--------|----------|
| 命名规范 | camelCase + PascalCase 一致: 10分, 少量偏差: 7分, 不一致: 3分 |
| JavaDoc 完整 | 公共类/方法都有 Doc: 10分, 部分: 6分, 无: 3分 |
| 无冗余代码 | 无重复代码: 10分, 少量重复: 7分, 明显复制粘贴: 3分 |
| 日志使用 | 使用 LOG 框架且级别合适: 10分, 使用 System.out: 3分 |

### 6. 错误处理 (Error Handling)

**权重**: 0.10 | **阈值**: 7/10

| 检查项 | 评分标准 |
|--------|----------|
| 异常不吞没 | 所有异常有处理: 10分, catch 后仅 log: 6分, 空 catch: 0分 |
| 用户友好的错误信息 | 信息包含原因和建议: 10分, 仅原因: 7分, 通用信息: 4分 |
| 资源管理 | try-with-resources: 10分, 手动 close: 7分, 可能泄露: 2分 |
| 正确的异常类型 | AnalysisException/DdlException 等分类正确: 10分, 通用 Exception: 4分 |

---

## 综合评分公式

```
综合分 = Σ(维度分 × 权重)

通过条件:
1. 编译正确性 = 10/10 (硬性)
2. 综合分 ≥ 7.5/10
3. 所有维度 ≥ 各自阈值
```
