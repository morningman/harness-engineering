# Doris BE (C++) 代码质量标准

> 用于 Evaluator Agent 评估 BE 侧 C++ 代码修改的质量

---

## 评分维度

### 1. 编译正确性 (Compilation Correctness)

**权重**: 0.30 | **阈值**: 10/10 (不可通融)

| 检查项 | 命令 | 通过条件 |
|--------|------|----------|
| 编译通过 | `cd be && ./build.sh --be` | 返回成功，无编译错误 |
| 无新增编译警告 | 编译输出检查 | 不引入新的 `-Wall` 警告 |
| Clang-format 通过 | `clang-format --dry-run -Werror` | 无格式化差异 |

### 2. 单元测试 (Unit Tests)

**权重**: 0.25 | **阈值**: 8/10

| 检查项 | 评分标准 |
|--------|----------|
| 现有 GTest 无回归 | 所有原有测试仍通过: 10分 |
| 新增代码有 GTest 覆盖 | 覆盖率 > 80%: 10分, > 60%: 8分, > 40%: 6分, < 40%: 4分 |
| 边界条件 | buffer 溢出 / null / 空输入测试: 10分, 部分: 7分, 无: 4分 |
| 内存安全 | ASAN/TSAN 通过: 10分, 未测试但评估安全: 7分 |

### 3. 内存管理 (Memory Management)

**权重**: 0.15 | **阈值**: 8/10

| 检查项 | 评分标准 |
|--------|----------|
| RAII 资源管理 | smart pointer / RAII: 10分, 手动 new/delete 但无泄漏: 6分, 有泄漏风险: 2分 |
| Memory Tracker 集成 | 正确使用 MemTracker: 10分, 部分使用: 7分, 未使用: 4分 |
| JNI 内存管理 | JNI 引用正确释放: 10分, 有 GlobalRef 泄漏风险: 3分 |
| 大内存分配 | 使用内存池 / 预分配: 10分, 频繁小分配: 6分, 无限制增长: 2分 |

### 4. 性能影响 (Performance Impact)

**权重**: 0.10 | **阈值**: 7/10

| 检查项 | 评分标准 |
|--------|----------|
| 热路径无性能退化 | benchmark 无退化: 10分, < 5% 退化: 7分, > 5%: 4分 |
| 锁使用 | lock-free 或 read-write lock: 10分, mutex 但粒度合理: 7分, 全局锁: 3分 |
| 向量化友好 | 列式处理 / SIMD: 10分, 行式但合理: 7分, 性能关键路径行式: 3分 |
| JNI 调用频率 | 批量调用: 10分, 合理频率: 7分, 每行都 JNI 调用: 2分 |

### 5. 架构对齐 (Architecture Alignment)

**权重**: 0.10 | **阈值**: 7/10

| 检查项 | 评分标准 |
|--------|----------|
| Pipeline / Operator 模式 | 正确使用 Pipeline 框架: 10分, 轻微偏差: 7分, 绕过框架: 3分 |
| Status 返回值 | 使用 Status 错误处理: 10分, 部分使用: 7分, 使用异常: 3分 |
| 线程安全 | 线程安全设计: 10分, 有锁保护: 7分, 有竞态风险: 2分 |
| 代码位置 | 文件放在正确的目录: 10分, 合理但不精确: 7分, 不合理: 3分 |

### 6. 代码风格 (Code Style)

**权重**: 0.05 | **阈值**: 6/10

| 检查项 | 评分标准 |
|--------|----------|
| 命名规范 | snake_case + PascalCase 一致: 10分, 少量偏差: 7分, 不一致: 3分 |
| Doxygen 注释 | 公共类/函数有文档: 10分, 部分: 6分, 无: 3分 |
| 头文件 | `#pragma once` + include 有序: 10分, 部分: 7分 |
| const 正确性 | const 使用合适: 10分, 遗漏部分 const: 7分, 明显错误: 3分 |

### 7. 错误处理 (Error Handling)

**权重**: 0.05 | **阈值**: 7/10

| 检查项 | 评分标准 |
|--------|----------|
| Status 传播 | RETURN_IF_ERROR 等宏正确使用: 10分, 手动检查但正确: 7分, 忽略 Status: 0分 |
| LOG 使用 | LOG(WARNING/ERROR) 级别合理: 10分, 全用 INFO: 5分 |
| 优雅降级 | 非致命错误不 crash: 10分, 大部分处理: 7分, 直接 abort: 2分 |

---

## 综合评分公式

```
综合分 = Σ(维度分 × 权重)

通过条件:
1. 编译正确性 = 10/10 (硬性)
2. 综合分 ≥ 7.5/10
3. 所有维度 ≥ 各自阈值
```

## 特殊规则

### JNI 相关修改额外检查

如果修改涉及 JNI 代码（`JniScanner`, `JniConnector`, `JniWriter`），额外检查：
- [ ] JNI 全局引用有正确释放
- [ ] Java 异常有检查和处理 (`env->ExceptionCheck()`)
- [ ] 大数组传递使用 `DirectByteBuffer` 而非 `byte[]`
- [ ] JNI 调用批量化（不在循环内单行调用）

### Scanner 相关修改额外检查

如果修改涉及 Scanner（`JniScanner`, `VScanner`, `VScanNode`），额外检查：
- [ ] `open()` → `get_next()` → `close()` 生命周期正确
- [ ] 支持 predicate pushdown（如适用）
- [ ] 支持 column pruning
- [ ] Scanner 线程安全（可被多个 pipeline task 并发调用）
