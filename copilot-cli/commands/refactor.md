# /refactor - Doris 代码重构

启动一个结构化的 Doris 代码重构流程。

## 流程

1. **分析现状**: 理解当前代码结构和问题
2. **定义目标**: 明确重构后的目标架构/模式
3. **建立基线**: 记录当前通过的测试列表
4. **制定计划**: 分解为最小安全步骤
5. **增量执行**: 每步完成后编译+测试验证
6. **功能等价验证**: 与基线对比，确保无回归

## 重构原则

- **最小改动**: 每步只做一件事（改名/移动/替换）
- **编译不断**: 每步完成后系统可编译
- **测试不红**: 每步完成后所有测试仍通过
- **可回滚**: 每步都有 git commit，可以随时回退

## 常见重构模式

### Pattern 1: 继承 → 组合
```
Step 1: 创建新的组合类
Step 2: 在组合类中实现逻辑
Step 3: 添加组合类的测试
Step 4: 逐个切换调用方
Step 5: 清理旧的继承类
```

### Pattern 2: Scanner 迁移 (JniScanner)
```
Step 1: 创建新 JniScanner 子类
Step 2: 实现 open/getNext/close
Step 3: 迁移数据读取逻辑
Step 4: 更新 ScanNode 使用新 Scanner
Step 5: 清理旧 Scanner 代码
```

### Pattern 3: Catalog 架构升级
```
Step 1: 定义新的 Catalog 接口/抽象类
Step 2: 实现新接口
Step 3: 添加兼容适配器
Step 4: 迁移调用方
Step 5: 移除适配器和旧代码
```
