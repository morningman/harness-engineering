# /plan - 制定 Doris 开发计划

请根据以下需求，为 Apache Doris 开发制定一份结构化的开发计划。

## 流程

1. **理解需求**: 分析用户的需求描述，识别功能点和技术挑战
2. **分析影响范围**: 确定涉及 FE/BE/Connector/文档的哪些部分
3. **加载模块约定**: 发现并读取相关模块的 `AGENTS.md` 文件
   ```bash
   find . -name "AGENTS.md" -type f | sort
   ```
   根据影响范围筛选相关的 AGENTS.md，提取约束作为计划的输入
4. **任务分解**: 将需求拆分为可管理的增量任务（每个任务 < 500 行修改）
5. **依赖分析**: 标注任务间的前后依赖关系
6. **添加验证步骤**: 每个任务包含具体的编译和测试验证命令
7. **产出计划文件**: 保存为 `plans/YYYY-MM-DD-plan-name.md`

## 计划模板

```markdown
# 开发计划: [功能名称]

## 需求概述
[一段话描述需求]

## 影响范围
- FE: [受影响的模块]
- BE: [受影响的模块]
- Connector: [受影响的Connector]
- 文档: [需要更新的文档]

## 任务分解

### Task 1: [任务名称]
- 描述: [做什么]
- 文件: [涉及的文件]
- 依赖: [前置任务]
- 验证:
  - 编译: `[命令]`
  - 测试: `[命令]`
- 预估改动: ~N 行

### Task 2: ...

## 风险点
1. [潜在风险和应对策略]

## 时间线
- Task 1: [预估]
- Task 2: [预估]
- 总计: [总预估]
```

## 注意事项
- 参考 Doris 根目录 AGENTS.md 中的项目知识
- **必须加载并遵守相关模块的 AGENTS.md 约定**
- 每个 Task 应该是一个可独立编译和测试的增量单元
- 优先级: 核心功能 → 边界处理 → 性能优化 → 文档
